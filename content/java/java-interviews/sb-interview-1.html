Q1. What is Spring Boot and how is it different from the traditional Spring Framework?

Spring Boot is an opinionated framework built on top of the Spring Framework that reduces boilerplate configuration and simplifies application development. Traditional Spring applications require manual configuration of beans, dependency management, web servers, and application contexts. Spring Boot automates these tasks using Auto Configuration, Starter Dependencies, and Embedded Servers. Key advantages include: - Auto Configuration - Starter Dependencies - Embedded Tomcat/Jetty/Undertow - Production-ready features - Minimal XML or Java configuration
Java 

@SpringBootApplication
public class MyApp {

    public static void main(String[] args) {
        SpringApplication.run(MyApp.class, args);
    }
}

Q2. What does @SpringBootApplication do? What three annotations does it combine?

`@SpringBootApplication` is a meta-annotation that combines three important annotations: `@SpringBootConfiguration`, `@EnableAutoConfiguration`, and `@ComponentScan`. Together, they bootstrap a Spring Boot application by enabling bean definitions, automatic configuration based on dependencies, and component scanning in the base package.
Java 

@SpringBootApplication
public class Application {

    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

Q3. What is Spring Boot Auto-Configuration and how does it work?

Auto-configuration automatically configures Spring beans based on the classpath, existing beans, and application properties. It uses conditional annotations like `@ConditionalOnClass`, `@ConditionalOnMissingBean`, and `@ConditionalOnProperty` to decide which configurations to apply.
Java 

@Configuration
@ConditionalOnClass(DataSource.class)
@ConditionalOnMissingBean(DataSource.class)
public class DataSourceAutoConfiguration {

    @Bean
    public DataSource dataSource() {
        return new H2DataSource();
    }
}

Q6. What is ApplicationContext in Spring?

Java

Q11. What are Spring Profiles? How do you activate them and use them for environment-specific configuration?

Spring Profiles allow you to define environment-specific configurations such as dev, test, staging, and production. You can create separate configuration files like `application-dev.yml` and `application-prod.yml`, and Spring Boot will load the active profile at runtime. You can also use `@Profile` annotation to conditionally load beans based on environment. Profiles can be activated using: - application.properties/yml - JVM argument (-Dspring.profiles.active=prod) - Environment variable (SPRING_PROFILES_ACTIVE)
Java 

@Profile("dev")
@Service
public class DevEmailService implements EmailService {

    public void send(String to, String msg) {
        System.out.println("Dev email sent to " + to);
    }
}

@Profile("prod")
@Service
public class ProdEmailService implements EmailService {

    public void send(String to, String msg) {
        // real email sending logic
    }
}

Q16. What is a RESTful API? How does Spring Boot support building one?

A RESTful API is an architectural style that uses HTTP methods to perform operations on resources. It is stateless, resource-based, and typically uses JSON for communication. Spring Boot supports REST APIs using Spring MVC with annotations like @RestController and request mapping annotations such as @GetMapping, @PostMapping, etc. Jackson automatically converts Java objects into JSON responses.
Java 

@RestController
@RequestMapping("/api/products")
public class ProductController {

    private final ProductService productService;

    public ProductController(ProductService productService) {
        this.productService = productService;
    }

    @GetMapping
    public List getAll() {
        return productService.findAll();
    }

    @GetMapping("/{id}")
    public Product getById(@PathVariable Long id) {
        return productService.findById(id);
    }

    @PostMapping
    @ResponseStatus(HttpStatus.CREATED)
    public Product create(@RequestBody @Valid Product product) {
        return productService.save(product);
    }

    @DeleteMapping("/{id}")
    @ResponseStatus(HttpStatus.NO_CONTENT)
    public void delete(@PathVariable Long id) {
        productService.delete(id);
    }
}

Q17. What is the difference between @PathVariable, @RequestParam, and @RequestBody?

@PathVariable extracts values from the URI path, @RequestParam extracts query parameters from the URL, and @RequestBody maps the entire HTTP request body (usually JSON) to a Java object. They are used in different parts of an HTTP request based on how data is sent.
Java 

@RestController
@RequestMapping("/api/users")
public class UserController {

    @GetMapping("/{id}")
    public User getById(@PathVariable Long id) {
        return userService.findById(id);
    }

    @GetMapping
    public List getAll(
            @RequestParam(defaultValue = "0") int page,
            @RequestParam(defaultValue = "10") int size,
            @RequestParam(required = false) String sort) {

        return userService.findAll(page, size, sort);
    }

    @PostMapping
    public User create(@RequestBody @Valid CreateUserRequest request) {
        return userService.create(request);
    }

    @PutMapping("/{id}")
    public User update(@PathVariable Long id,
                       @RequestParam(required = false) boolean notify,
                       @RequestBody UpdateUserRequest request) {

        return userService.update(id, request, notify);
    }
}

Q18. What is ResponseEntity? Why is it used?

ResponseEntity gives full control over HTTP response, including status code, headers, and body. It is used when we need to return proper REST semantics like 200 OK, 201 Created, 204 No Content, or custom headers.
Java 

@RestController
@RequestMapping("/api/orders")
public class OrderController {

    @GetMapping("/{id}")
    public ResponseEntity getOrder(@PathVariable Long id) {
        return orderService.findById(id)
                .map(ResponseEntity::ok)
                .orElse(ResponseEntity.notFound().build());
    }

    @PostMapping
    public ResponseEntity create(@RequestBody Order order,
                                        UriComponentsBuilder uriBuilder) {

        Order saved = orderService.create(order);

        URI location = uriBuilder
                .path("/api/orders/{id}")
                .buildAndExpand(saved.getId())
                .toUri();

        return ResponseEntity.created(location).body(saved);
    }

    @DeleteMapping("/{id}")
    public ResponseEntity delete(@PathVariable Long id) {
        orderService.delete(id);
        return ResponseEntity.noContent().build();
    }
}

Q19. What is global exception handling in Spring Boot?

Global exception handling is used to handle exceptions centrally across all controllers using @ControllerAdvice or @RestControllerAdvice. It ensures consistent error responses and removes the need for try-catch blocks in controllers.
Java 

@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(ResourceNotFoundException.class)
    @ResponseStatus(HttpStatus.NOT_FOUND)
    public Map handleNotFound(ResourceNotFoundException ex) {

        Map error = new HashMap<>();
        error.put("message", ex.getMessage());
        error.put("status", 404);

        return error;
    }

    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public Map handleGeneric(Exception ex) {

        Map error = new HashMap<>();
        error.put("message", "Internal Server Error");
        error.put("status", 500);

        return error;
    }
}

Q21. What is RestTemplate and WebClient? Which should you use in modern Spring Boot?

RestTemplate is a synchronous (blocking) HTTP client used to call external REST APIs. WebClient is the modern reactive (non-blocking) HTTP client introduced in Spring WebFlux. WebClient is recommended for new applications because it supports async processing and better scalability, while RestTemplate is in maintenance mode.
Java 

// RestTemplate (blocking)
RestTemplate restTemplate = new RestTemplate();

WeatherData data = restTemplate.getForObject(
    "https://api.weather.com/v1/city?name=London",
    WeatherData.class
);

// WebClient (non-blocking)
WebClient webClient = WebClient.create("https://api.weather.com");

Mono<WeatherData> response = webClient.get()
    .uri("/v1/city?name={city}", "London")
    .retrieve()
    .bodyToMono(WeatherData.class);

// Optional blocking
WeatherData data2 = response.block();

Q22. What is CORS and how do you configure it in Spring Boot?

CORS (Cross-Origin Resource Sharing) is a browser security mechanism that controls whether a frontend running on one domain can access APIs on another domain. Spring Boot allows CORS configuration at method level, globally using WebMvcConfigurer, or via Spring Security.
Java 

// Method-level CORS
@CrossOrigin(origins = "https://frontend.com")
@RestController
public class ApiController {

    @GetMapping("/public")
    public String hello() {
        return "open";
    }
}

// Global CORS configuration
@Configuration
public class WebConfig implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
            .allowedOrigins("https://frontend.com")
            .allowedMethods("GET", "POST", "PUT", "DELETE")
            .allowCredentials(true);
    }
}

Q23. What is Spring Transaction Management? How does @Transactional work?

Spring Transaction Management ensures that a group of database operations either fully succeed or fully fail. @Transactional uses AOP to start a transaction before method execution and commits it if successful. If a RuntimeException occurs, it automatically rolls back the transaction.

        return repository.findAll();
    }

    public void process() throws Exception {
        // custom rollback rules
    }
}

Q24. What is Spring Caching? How do @Cacheable, @CachePut, and @CacheEvict work?

Spring Caching allows storing method results to improve performance by avoiding repeated database or API calls. @Cacheable stores results, @CachePut updates the cache, and @CacheEvict removes entries. It supports providers like in-memory cache, Caffeine, and Redis.
Java 

@EnableCaching
@SpringBootApplication
public class App { }

@Service
public class ProductService {

    @Cacheable("products")
    public Product getById(Long id) {
        return repo.findById(id);
    }

    @CachePut(value = "products", key = "#product.id")
    public Product update(Product product) {
        return repo.save(product);
    }

    @CacheEvict(value = "products", key = "#id")
    public void delete(Long id) {
        repo.deleteById(id);
    }
}

Q25. What is @Async in Spring Boot and how does it work?

@Async enables asynchronous method execution by running methods in a separate thread pool managed by Spring. The caller does not wait for the method to complete. It works using Spring proxies and only functions when called from outside the same bean.
Java 

@EnableAsync
@SpringBootApplication
public class App { }

@Service
public class NotificationService {

    @Async
    public void sendEmail(String email) {
        System.out.println("Sending email...");
    }

    @Async
    public CompletableFuture<String> process() {
        return CompletableFuture.completedFuture("done");
    }
}

@Bean
public Executor taskExecutor() {
    ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
    executor.setCorePoolSize(5);
    executor.setMaxPoolSize(20);
    executor.initialize();
    return executor;
}

Q26. What is Spring's @Scheduled annotation? How do you set up scheduled tasks?

@Scheduled is used to run methods automatically at fixed intervals or based on cron expressions. It supports fixedRate, fixedDelay, and cron-based scheduling. You must enable scheduling using @EnableScheduling. It is commonly used for background jobs like polling APIs, cleanup tasks, or report generation.
Java 

// Enable scheduling
@SpringBootApplication
@EnableScheduling
public class Application { }

// Scheduled tasks
@Component
public class ScheduledTasks {

    // Runs every 5 seconds (fixed rate)
    @Scheduled(fixedRate = 5000)
    public void pollExternalApi() {
        System.out.println("Polling API...");
    }

    // Runs 10 seconds after previous execution finishes
    @Scheduled(fixedDelay = 10000)
    public void processQueue() {
        System.out.println("Processing queue...");
    }

    // Cron-based scheduling
    @Scheduled(cron = "0 30 2 * * *")
    public void dailyCleanup() {
        System.out.println("Running cleanup...");
    }
}

Q27. What is @SpringBootTest? When should you use it?

@SpringBootTest is used for integration testing in Spring Boot. It loads the full application context, including all beans and configurations. It is used when you want to test the complete flow of the application. It supports different web environments like MOCK (default), RANDOM_PORT, and NONE.

    private MockMvc mockMvc;

    @Test

        mockMvc.perform(get("/api/products/1"))
               .andExpect(status().isOk());

@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)

Q28. What is @MockBean and how is it different from @Mock?

@MockBean is used in Spring Boot tests to replace a Spring-managed bean with a Mockito mock inside the application context. @Mock is used in pure unit tests without Spring context. @MockBean is useful in integration or slice tests where you want to mock dependencies inside Spring.
Java 

// Pure unit test with Mockito
@ExtendWith(MockitoExtension.class)
class OrderServiceTest {

    @Mock
    private OrderRepository orderRepository;

    @InjectMocks
    private OrderService orderService;

    @Test
    void testPlaceOrder() {
        when(orderRepository.save(any()))
            .thenReturn(new Order(1L));

        Order result = orderService.place(new OrderRequest());
    }
}

// Spring test with mocked bean
@WebMvcTest(OrderController.class)
class OrderControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private OrderService orderService;
}

Q29. What is @WebMvcTest? How is it used?

@WebMvcTest is a Spring Boot test annotation used to test only the web layer (controllers). It loads only controller-related beans and mocks all service dependencies. It is faster than @SpringBootTest and is used for testing REST APIs in isolation.
Java 


@WebMvcTest(ProductController.class)
class ProductControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private ProductService productService;

    @Test
    void testGetProduct() throws Exception {
        mockMvc.perform(get("/api/products/1"))
               .andExpect(status().isOk());
    }

    @Test
    void testInvalidRequest() throws Exception {
        mockMvc.perform(post("/api/products")
               .contentType(MediaType.APPLICATION_JSON)
               .content("{}"))
               .andExpect(status().isBadRequest());
    }
}

Q30. What is the difference between unit testing and integration testing in Spring Boot?

Unit testing tests a single class in isolation using mocks and does not load Spring context. Integration testing loads the Spring context and tests how multiple components work together, often using a real or in-memory database.
Java 


// Unit test (Mockito only)
@ExtendWith(MockitoExtension.class)
class DiscountServiceTest {

    @Mock
    private UserRepository userRepository;

    @InjectMocks
    private DiscountService discountService;

    @Test
    void testDiscount() {
        when(userRepository.findById(1L))
            .thenReturn(Optional.of(new User(1L, true)));

        double discount = discountService.calculate(1L, 100.0);
    }
}

// Integration test (Spring context)
@SpringBootTest
@Transactional
class OrderIntegrationTest {

    @Autowired
    private OrderService orderService;

    void testOrderFlow() {
        Order order = orderService.place(new OrderRequest("Alice", 2));
    }
