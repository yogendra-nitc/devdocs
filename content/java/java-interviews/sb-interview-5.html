Spring Boot questions on REST APIs development

Q1. What is a REST API and what are its key principles?

REST (Representational State Transfer) is an architectural style for designing web services that communicate over HTTP. REST APIs expose resources using URIs and standard HTTP methods. Key principles of REST include: - Stateless communication - Client-server architecture - Resource-based URLs - Uniform interface - Cacheable responses - Layered system Spring Boot simplifies REST API development using annotations such as `@RestController`, `@RequestMapping`, and `@GetMapping`.
Java 

@RestController
@RequestMapping("/employees")
public class EmployeeController {

    @GetMapping
    public String getEmployees() {
        return "Employee List";
    }
}

Q2. What is @RestController annotation?

`@RestController` is a specialized Spring annotation used to create RESTful web services. It combines `@Controller` and `@ResponseBody`, meaning the return value of every handler method is written directly to the HTTP response body instead of resolving a view. It is the preferred annotation for REST APIs.
Java 

@RestController
public class WelcomeController {

    @GetMapping("/welcome")
    public String welcome() {
        return "Welcome to Spring Boot REST API";
    }
}

Q3. What is @RequestMapping annotation?

`@RequestMapping` maps HTTP requests to controller classes or methods. It can define the URL path, HTTP method, headers, content type, and accepted media type. It can be applied at both the class level and method level.
Java 

@RestController
@RequestMapping("/employees")
public class EmployeeController {

    @RequestMapping("/all")
    public String getEmployees() {
        return "All Employees";
    }
}

Q4. What is the difference between @Controller and @RestController?

`@Controller` is primarily used in Spring MVC applications to return views such as JSP or Thymeleaf templates. `@RestController` is used for REST APIs and automatically serializes Java objects into JSON or XML for the HTTP response. Use `@Controller` for web pages and `@RestController` for RESTful services.
Java 

@Controller
public class ViewController {

    @GetMapping("/home")
    public String home() {
        return "home";
    }
}

@RestController
public class ApiController {

    @GetMapping("/api/message")
    public String message() {
        return "Hello REST";
    }
}

Q5. What are the commonly used HTTP methods in REST APIs?

REST APIs primarily use the following HTTP methods: - GET – Retrieve resources - POST – Create a new resource - PUT – Update an existing resource completely - PATCH – Partially update a resource - DELETE – Remove a resource Spring Boot provides dedicated annotations for each HTTP method.
Java 

@GetMapping
public List<Employee> getEmployees() { }

@PostMapping
public Employee createEmployee() { }

@PutMapping("/{id}")
public Employee updateEmployee() { }

@PatchMapping("/{id}")
public Employee updateSalary() { }

@DeleteMapping("/{id}")
public void deleteEmployee() { }

Q6. What is @GetMapping annotation?

`@GetMapping` is a shortcut for `@RequestMapping(method = RequestMethod.GET)`. It maps HTTP GET requests to a controller method and is typically used to retrieve one or more resources.
Java 

@RestController
@RequestMapping("/employees")
public class EmployeeController {

    @GetMapping("/{id}")
    public String getEmployee() {
        return "Employee Details";
    }
}

Q7. What is @PostMapping annotation?

`@PostMapping` maps HTTP POST requests to a controller method. It is commonly used to create new resources in the application. The request body is usually received using the `@RequestBody` annotation.
Java 

@PostMapping
public Employee createEmployee(
        @RequestBody Employee employee) {

    return employee;
}

Q8. What are @PutMapping and @PatchMapping? What is the difference?

`@PutMapping` is used for complete updates of a resource. The client typically sends the entire resource representation. `@PatchMapping` is used for partial updates where only specific fields are modified. PUT replaces the resource, whereas PATCH modifies only the supplied fields.
Java 

@PutMapping("/{id}")
public Employee updateEmployee(
        @PathVariable Long id,
        @RequestBody Employee employee) {

    return employee;
}

@PatchMapping("/{id}")
public Employee updateSalary(
        @PathVariable Long id,
        @RequestBody Employee employee) {

    return employee;
}

Q9. What is @DeleteMapping annotation?

`@DeleteMapping` maps HTTP DELETE requests to a controller method. It is used to remove a resource identified by a unique identifier. Typically, a successful deletion returns HTTP status 204 (No Content) or 200 (OK).
Java 

@DeleteMapping("/{id}")
public void deleteEmployee(
        @PathVariable Long id) {

    System.out.println("Deleted Employee " + id);
}

Q10. What is @RequestBody annotation?

`@RequestBody` binds the JSON request payload to a Java object. Spring Boot automatically converts the incoming JSON into a Java object using the Jackson library. It is commonly used with POST, PUT, and PATCH requests.
Java 

@PostMapping("/employees")
public Employee saveEmployee(
        @RequestBody Employee employee) {

    return employee;
}

class Employee {

    private Long id;
    private String name;

    // Getters and Setters
}

Q11. What is @PathVariable annotation?

`@PathVariable` is used to extract values from the URI path and bind them to method parameters. It is commonly used to identify a specific resource in REST APIs. For example, in `/employees/101`, the value `101` can be mapped to a method parameter using `@PathVariable`.
Java 

@RestController
@RequestMapping("/employees")
public class EmployeeController {

    @GetMapping("/{id}")
    public String getEmployeeById(
            @PathVariable Long id) {

        return "Employee Id : " + id;
    }
}

Q12. What is @RequestParam annotation?

`@RequestParam` binds query parameters from the URL to method parameters. It is commonly used for filtering, searching, sorting, and pagination. For example: `GET /employees?department=IT`
Java 

@GetMapping("/search")
public String searchEmployee(
        @RequestParam String department) {

    return "Department : " + department;
}

Q13. What is @RequestHeader annotation?

`@RequestHeader` is used to read values from HTTP request headers. It is commonly used to retrieve headers such as `Authorization`, `Content-Type`, `Accept`, or custom headers.
Java 

@GetMapping("/profile")
public String getProfile(
        @RequestHeader("Authorization")
        String token) {

    return token;
}

Q14. What is ResponseEntity and why is it used?

`ResponseEntity` represents the complete HTTP response, including the response body, HTTP status code, and headers. It provides greater control over the response compared to returning an object directly. It is widely used in production REST APIs.
Java 

@GetMapping("/{id}")
public ResponseEntity<String> getEmployee(
        @PathVariable Long id) {

    return ResponseEntity
            .ok("Employee Found");
}

Q15. What is @ResponseStatus annotation?

`@ResponseStatus` specifies the HTTP status code returned by a controller method or exception class. It improves readability by explicitly defining the expected response status.
Java 

@PostMapping
@ResponseStatus(HttpStatus.CREATED)
public Employee createEmployee(
        @RequestBody Employee employee) {

    return employee;
}

Q16. What is @ExceptionHandler annotation?

`@ExceptionHandler` is used to handle exceptions thrown by controller methods. It allows applications to return meaningful error messages and HTTP status codes instead of exposing stack traces. It can be defined within a controller or in a global exception handler.
Java 

@RestController
public class EmployeeController {

    @ExceptionHandler(RuntimeException.class)
    public ResponseEntity<String> handleException(
            RuntimeException ex) {

        return ResponseEntity
                .badRequest()
                .body(ex.getMessage());
    }
}

Q17. What is @ControllerAdvice annotation?

`@ControllerAdvice` provides centralized exception handling across all controllers. Instead of writing `@ExceptionHandler` methods in every controller, a global handler can manage exceptions for the entire application. It promotes code reuse and consistent error responses.
Java 

@ControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(Exception.class)
    public ResponseEntity<String> handleException(
            Exception ex) {

        return ResponseEntity
                .internalServerError()
                .body(ex.getMessage());
    }
}

Q18. What is REST API Versioning and why is it important?

REST API Versioning allows multiple versions of an API to coexist without breaking existing clients. It enables backward compatibility when introducing new features or modifying API behavior. Common approaches include: - URI Versioning (`/v1/employees`) - Query Parameter Versioning - Custom Header Versioning - Media Type Versioning URI versioning is the most commonly used approach.
Java 

@RestController
@RequestMapping("/api/v1/employees")
public class EmployeeController {

    @GetMapping
    public String getEmployees() {

        return "Version 1";
    }
}

Q19. What is Content Negotiation in Spring Boot?

Content Negotiation is the process of selecting the response format based on the client's request. Spring Boot supports multiple response formats such as JSON and XML. The client specifies the desired format using the `Accept` header.
Java 

@GetMapping(
    value = "/employees",
    produces = MediaType.APPLICATION_JSON_VALUE
)
public List<Employee> getEmployees() {

    return List.of();
}

Q20. What is @Valid annotation and how is it used for request validation?

`@Valid` triggers validation of incoming request data using Jakarta Bean Validation annotations such as `@NotNull`, `@NotBlank`, `@Email`, `@Min`, and `@Max`. If validation fails, Spring Boot returns a validation error response before executing the controller method.
Java 

public class Employee {

    @NotBlank
    private String name;

    @Email
    private String email;

    // Getters and Setters
}

@PostMapping
public Employee createEmployee(
        @Valid @RequestBody Employee employee) {

    return employee;
}
